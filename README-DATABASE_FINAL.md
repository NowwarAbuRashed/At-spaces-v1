## 8) Soft Delete Policy âś… (YES)
We use **soft delete** to preserve history and avoid data loss.

Recommended fields (example):
- `deleted_at TIMESTAMP NULL`
- Optionally: `deleted_by INT NULL`

Soft delete should apply to most business tables:
- users, branches, vendor_services, bookings, payments, approval_requests, conversations, messages, etc.

## 11) Primary Image Rule (Strict)
Goal:
- Only **one** primary image (`is_primary=true`) per:
  - service in `service_images`
  - vendor service in `vendor_service_images`

Implementation:
- Use **partial unique index** (filtered uniqueness):
  - Unique only when `is_primary=true`

PostgreSQL supports partial indexes:
- https://www.postgresql.org/docs/current/indexes-partial.html

Prisma note:
- Some advanced DB features may require custom SQL in migrations.
- https://www.prisma.io/docs/orm/prisma-migrate/workflows/unsupported-database-features

Example (PostgreSQL):
```sql
CREATE UNIQUE INDEX service_images_one_primary
ON service_images(service_id)
WHERE is_primary = true;
```
